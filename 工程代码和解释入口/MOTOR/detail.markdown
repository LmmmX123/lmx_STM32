## 1. 系统概述与设计目标
本档案记录基于 `STM32F407ZGT6` 主控与 `HBT42C` 一体式闭环驱动器的四轴步进电机控制固件设计逻辑、封装规范、时序合规性及调试流程。固件严格遵循《HBT42C V1.0 使用说明书》电气指标、信号时序图与接线规范，采用**实例化硬件抽象层 + 共享使能管理 + 固定安全参数**的架构，实现单端共阴极接线模式下的低振动、高精度、防失步闭环运动控制。

**核心设计约束**：
- 硬件资源限制：仅使用 `PB3~PB7` 与 `PD0~PD3`，`PB3` 共享为全局使能线。
- 控制逻辑简化：细分与转速参数固化，调用接口仅保留 `电机ID` 与 `方向`。
- 时序强合规：软件延时强制覆盖手册 `t1~t4` 时序要求，脉冲频率锁定于安全区间。
- 运行模式：阻塞式梯形加减速，保证单轴定位精度 `±1 Pulse`（手册 §2.1 指标）。

## 2. 硬件接口映射与电气规范（严格对照手册 §3.1/§3.5）

### 2.1 引脚映射表（共阴极单端接法）
| 信号 | 驱动器端子 | MCU 引脚 | 电气特性与接线要求 |
|------|------------|----------|-------------------|
| 共享使能 | `EN+` / `EN-` | `PB3` / `GND` | 手册 §3.2/FAQ 明确：高电平或悬空为使能有效，低电平关闭。代码统一驱动 `GPIO_PIN_SET`。 |
| 脉冲1~4 | `PU+` / `PU-` | `PB4~7`, `PD0~2` / `GND` | 逻辑电平兼容 `4.5~24V`，输入电流 `10~16mA`。**无需串联限流电阻**（手册 §3.2）。 |
| 方向1~4 | `DR+` / `DR-` | `PB5~7`, `PD1~3` / `GND` | 同上。高电平=`CW`，低电平=`CCW`（可通过 `SW5` 拨码软反转）。 |
| 报警输出 | `AL+` / `AL-` | 未接入 | 集电极开路 `OC` 输出，最大上拉 `24V`，最大电流 `100mA`。预留扩展。 |

### 2.2 抗干扰与布线强制规范（手册 §3.5）
1. **屏蔽接地**：控制信号必须采用屏蔽线，屏蔽层**仅 MCU 端单端接地**，驱动器端悬空。同机仅允许单点接地。
2. **走线隔离**：脉冲/方向信号线与 `24V` 电源线间距 `≥10cm`，禁止并排捆扎，防止电源噪声耦合导致定位偏差。
3. **供电拓扑**：多电机共用电源时，必须在电源端子处**并联连接**，禁止链状串联。
4. **操作禁令**：严禁带电插拔端子；严禁导线加锡后压接（接触电阻增大会导致端子过热损坏）。

## 3. 控制时序合规性分析（对照手册 §3.3 图5）

驱动器内部采用 `DSP+FOC/SVPWM` 闭环算法，对外部时序有严格要求。固件通过软件延时强制满足以下规范：

| 时序参数 | 手册要求 | 固件实现策略 | 安全余量 |
|----------|----------|--------------|----------|
| `t1` (ENA建立时间) | `ENA` 需提前 `DIR` `≥5μs` 确定为高 | `motor_global_enable()` 后执行 `delay_us(10)` | `+5μs` |
| `t2` (DIR建立时间) | `DIR` 需提前 `PUL` 下降沿 `≥1μs` 确定状态 | `set_motor_dir()` 后执行 `delay_us(5)` | `+4μs` |
| `t3` (脉冲高电平宽度) | `≥1.5μs` | `pulse_out()` 强制 `half_period ≥ 2μs` | `+0.5μs` |
| `t4` (脉冲低电平宽度) | `≥1.5μs` | 同上，高低电平对称输出 | `+0.5μs` |
| 逻辑电平阈值 | 高 `>3.5V`，低 `<0.5V` | STM32 `3.3V` 推挽输出完全满足 | `-` |

> 📌 **注**：手册 §3.4 注明“单/双脉冲模式、脉冲有效沿可通过PC软件设置”。本固件默认使用工厂出厂标准（单脉冲模式），上升/下降沿触发均兼容。

## 4. 软件架构与函数封装逻辑

### 4.1 封装设计原则
1. **去参数化**：将 `400 Pulse/rev`（拨码默认）与 `60 RPM`（`4kHz` 安全频率）固化，降低调用复杂度。
2. **实例化映射**：弃用硬编码引脚，采用 `MotorInst_t` 结构体数组。硬件拓扑与驱动逻辑解耦，便于后期PCB改版。
3. **共享使能策略**：4轴共用 `PB3`。牺牲单轴独立脱机能力，换取GPIO节省与布线简化。适用于顺序测试或协同定位场景。
4. **阻塞式梯形轮廓**：主循环内完成脉冲生成与频率微调。保证400步精准走完，无状态机复杂度，适合初期验证。

### 4.2 数据结构定义
```c
typedef struct {
    GPIO_TypeDef* port;      // 所属GPIO端口 (GPIOB/GPIOD)
    uint16_t pin_pul;        // 脉冲输出引脚
    uint16_t pin_dir;        // 方向控制引脚
    volatile bool is_busy;   // 运行状态标志 (预留RTOS/中断调度)
} MotorInst_t;
```
数组 `motors[4]` 严格按硬件分配映射。`is_busy` 标记当前轴是否处于运动阻塞期，便于上层状态机查询。

## 5. 核心代码实现与时序映射详解

### 5.1 `delay_us(uint32_t nus)`
- **校准原理**：160MHz 主频下，1时钟周期=`6.25ns`。`while(cycles--)` 单次迭代约消耗 `5` 个周期（递减+比较+跳转+取指）。校准系数 `160/5 ≈ 32`。
- **防优化**：`__IO` (`volatile`) 强制编译器保留循环体，确保微秒级延时精度 `±10%`。满足时序控制底线。

### 5.2 `motor_global_enable(bool en)`
- **作用**：统一控制 `PB3`。手册 FAQ 明确指出“使能信号为低→拉高或悬空即可”。代码驱动高电平使能。
- **时序**：在发脉冲前调用，配合 `delay_us(10)` 覆盖 `t1` 要求，避免 `DIR` 切换时电机未就绪。

### 5.3 `set_motor_dir(MotorInst_t* m, bool cw)`
- **作用**：写入目标电机的 `DR` 引脚。高低电平对应 `CW/CCW`。
- **时序**：写入后延时 `5μs`，严格覆盖 `t2` (`DIR`提前`PUL`下降沿`≥1μs`)，防止脉冲边沿与方向跳变重叠导致驱动器误判。

### 5.4 `pulse_out(MotorInst_t* m, uint32_t half_us)`
- **作用**：生成单个完整脉冲（高+低）。
- **防护**：强制 `half_us < 2 ? 2`，确保 `t3/t4 ≥ 1.5μs`。直接调用 `HAL_GPIO_WritePin`，无中断干扰下时序确定。

### 5.5 `motor_run(uint8_t id, bool cw)`
- **输入校验**：`id >= 4` 直接返回，防数组越界。
- **参数固化**：
  - `total_steps = 400`（1转，匹配拨码默认）
  - `start_freq = 500 Hz`（防低频共振最低启动频率）
  - `target_freq = 4000 Hz`（`60 RPM`，远低于手册 `200kHz` 默认上限）
  - `accel_steps = 150`, `dec_start = 250`（150步加速→100步匀速→150步减速）
- **加减速算法**：线性频率斜坡。加速度计算：`0→60 RPM` 耗时约 `75ms`，实际加速度 `≈0.8 RPM/ms`，远低于手册空载极限 `100 RPM/ms`（§2.1），确保不堵转、不超调。
- **主循环**：严格发满 `400` 个脉冲后退出，更新 `is_busy = false`。位置误差控制精度达手册指标 `±1 Pulse`。

## 7. 调试流程与故障排查指南

### 7.1 LED 状态判读（手册 §3.1.4 / §5）
| 绿灯 | 红灯 | 含义 | 处理措施 |
|------|------|------|----------|
| 常亮 | 熄灭 | 电源正常，待机 | 可发送脉冲 |
| 常亮 | 闪烁1次 (2Hz) | 过流故障 (`I_peak ≥ 6A`) | 检查负载卡滞/短路，**断电重启** |
| 常亮 | 闪烁2次 (2Hz) | 过压故障 (`Vdc ≥ 60V`) | 测量电源，**断电重启** |
| 常亮 | 闪烁7次 (2Hz) | 跟踪误差超差故障 | 检查细分拨码/信号干扰，**断电重启** |
> ⚠️ **重要**：手册 §5 明确指出，所有保护触发后必须**断电后重新上电**方可清除故障，软件无法直接复位。

### 7.2 常见问题对照表（手册 §6.1）
| 现象 | 可能原因 | 排查动作 |
|------|----------|----------|
| 电机不转，绿灯亮 | `ENA` 为低或未接线 | 用万用表测 `PB3` 是否为 `3.3V`；或手动悬空 `EN+` 测试 |
| 只朝一个方向转 | `DIR` 接线反/逻辑反 | 对调 `DR+/DR-`，或修改 `cw` 传参逻辑 |
| 走位不准/丢步 | 屏蔽未单端接地/频率过高 | 检查屏蔽层仅MCU端接地；降低 `target_freq` 测试 |
| 高速堵转/异响 | 加速过快/电压偏低 | 增大 `accel_steps`；检查电源是否稳定 `24V` |
| 红灯闪烁报警 | 内部保护触发 | 记录闪烁次数，按 §5 断电重启 |

### 7.3 仪器验证建议
- **逻辑分析仪**：抓取 `PU/DIR` 波形，验证 `t2`（`DIR`稳定早于`PUL`下降沿）、`t3/t4`（脉宽`≥1.5μs`）。
- **示波器**：观察 `PU` 边沿是否陡峭（无振铃/过冲），确认 `GPIO_SPEED_FREQ_HIGH` 生效。

## 8. 版本管理与后续扩展路径
- **V1.0**：基础阻塞式控制，4轴顺序运行，共享使能，参数固化。满足单轴验证与简单多轴定位。
- **后续升级路径**：
  1. **四轴同步运行**：将 `pulse_out` 移至 `TIMx` 更新中断，状态机改为非阻塞调度。
  2. **实时调速/多圈运行**：暴露 `speed_rpm` 与 `steps` 参数，动态重算 `target_freq` 与加减速斜率。
  3. **故障自愈与报警监测**：接入 `AL+` 引脚中断，监测报警信号并触发安全停机流程。
  4. **细分/方向软配置**：通过串口/EEPROM 动态覆盖拨码默认值，实现免跳线配置。
