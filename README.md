# 工创项目学习资料 README
面向：智能物流赛道
为了组内的成员的STM32成员的STM32软件及部分有限知识的资源分享，主要核心内容包括STM32（不限版本）和联合CUBEMX的HAL库开发的学习、复习笔记***仅代表个人***

##  项目说明
本仓库为**工创智能物流项目**配套学习与开发资料合集，基于本人第一学期STM32开发、视觉处理、机械结构设计、通信协议等实战经验整理而成，全部开源共享，方便队友快速上手、查漏补缺、协同开发。资料由浅入深、兼顾入门与实战，适合零基础起步，也可作为项目全流程开发的核心参考手册。

---

##  文件夹/文件详细说明
### 1. `CUBEMX+KEIL开发的STM32相关`
- **定位**：STM32开发核心入门与实战资料，零基础友好
- **核心内容**：依托B站**铁头山羊老师**STM32课程整理的学习笔记、开发心得、关键知识点、常见问题排查方案，覆盖CubeMX配置、Keil工程搭建、HAL库开发全流程
- **重点学习指南**：
  - 入门阶段：先跟着课程+本文件夹笔记，完成STM32基础外设（GPIO、定时器、串口等）的开发实践
  - 进阶阶段：深入理解通信协议、时钟、UART等核心原理，为项目功能开发打牢基础
  - 核心感悟：只懂原理≠掌握，跟着例程跑、会改参数是入门，能看懂数据手册、自定义底层操作才是真正精通
- **配套课程链接**：https://www.bilibili.com/video/BV16J4m1w7HB

### 2. `C语言相关-应试为主`
- **定位**：C语言基础复习与应试资料***注意:只是应试的***
- **用途**：快速回顾C语言核心语法、指针、结构体、函数等基础知识点，适合开发中遇到C语言基础问题时随时查阅，补全底层编程能力，为STM32代码开发筑牢根基

### 3. `F407ZGT6板载资源`
- **说明**：项目核心硬件**鹿小班F407ZGT6开发板**全套配套资料，ZGT6/ZET6版本通用
- **包含内容**：开发板原理图、用户手册、原厂例程、驱动文件、固件包等完整资源
- **资料网盘（3.29更新）**：
  - 链接：https://pan.baidu.com/s/1S2JqMykTJUSsO8aP0qzsZQ
  - 提取码：1234
- **使用建议**：硬件开发、引脚分配、库函数调用遇到问题时，优先查阅本文件夹原厂资料，快速定位解决方案

### 4. `视觉相关`
- **定位**：视觉方向补充学习资料，不定期更新
- **内容**：OPENMV基础入门、OPENCV核心知识点、图像处理实战技巧、视觉识别算法笔记
- **使用建议**：项目涉及物料识别、视觉定位、路径检测等视觉相关任务时参考，有OPENMV/OPENCV开发问题可随时沟通

### 5. `硬件的相关知识`
- **定位**：通信协议与硬件底层深度笔记
- **核心内容**：包含IIC、SPI等常用通信协议的手写整理笔记、原理详解、实战代码、常见问题排查
- **使用建议**：项目中传感器通信、外设驱动开发时重点参考，手写笔记干货密集，看不懂可随时沟通或借助AI辅助理解

### 6. `车的机械结构`
- **定位**：智能物流搬运小车机械设计全流程资料
- **包含内容**：
  - `智能物流搬运小车`：小车完整三维建模文件
  - `智能物流搬运小车step`：STEP格式通用模型文件，适配各类CAD软件
  - `detail.txt`：机械结构核心说明文档
- **核心重点**：文件夹内**小车组装建议WORD文档**为核心参考，详细说明小车结构设计、组装流程、安装注意事项、结构优化建议，是机械组装环节的核心指南，务必重点阅读
- **使用建议**：机械结构设计、零件加工、小车组装阶段全程参考，确保结构设计合理、组装流程规范

### 7. `W2-李铭萱`（技术档案）
- **定位**：项目核心技术档案，个人开发全流程记录
- **内容**：涵盖项目需求分析、技术方案选型、硬件电路设计、软件开发流程、调试记录、问题总结、优化方案等完整技术文档
- **重要性**：本文件为项目开发的核心技术沉淀，完整记录了项目从0到1的全流程思路与细节，是队友理解项目整体架构、协同开发、快速上手的核心参考，务必重点研读

##  使用建议（给队友）
1. **入门阶段**：优先学习`CUBEMX+KEIL开发的STM32相关`+配套课程，补全`C语言相关-应试为主`基础，打牢STM32开发能力
2. **硬件开发**：以`F407ZGT6板载资源`为核心参考，结合`硬件的相关知识`中的通信协议笔记，完成外设驱动开发
3. **机械结构**：小车设计与组装阶段，重点阅读`车的机械结构`中的组装建议WORD文档，参考建模文件完成结构优化
4. **视觉开发**：涉及视觉任务时，查阅`视觉相关`笔记，结合OPENMV硬件完成功能开发
5. **项目协同**：全程以`W2-李铭萱`技术档案为核心参考，理解项目整体架构，确保开发方向一致
6. **问题解决**：遇到问题先查对应文件夹资料、动手调试，再沟通解决，在实践中持续精进
